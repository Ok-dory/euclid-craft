# 유클리드 크래프트 — 프로젝트 플랜

## 프로젝트 개요
- 한줄 설명: 유클리드 원론 1권의 공리·공준을 조합(크래프팅)해서 정리를 발견하는 수학 웹게임
- 타겟 유저: 수학에 흥미 있는 10~30대, 퍼즐·크래프팅 게임 유저, 모에 문화 친숙한 20대
- 핵심 가치 제안: 공리 10개에서 시작해 48개 정리를 직접 손으로 발견하는 크래프팅 경험

## 기능 명세

### MVP (1차 구현)
- 크래프팅 보드: 공리/정리 카드 1~10장 선택 → 조합 → 새 정리 언락 — 우선순위: 높음
- 조합 로직: 유클리드 원론 1권 기반 조합 레시피 (공리/공준 5+5 → 정리 48개) — 우선순위: 높음
- 도감(컬렉션): 발견한 정리 카드 목록, 설명, 언락 조건 확인 — 우선순위: 높음
- 진행 저장: 언락 상태 로컬스토리지 저장, 자동 저장 — 우선순위: 높음
- 캐릭터 의인화: 각 카드에 고유 캐릭터 슬롯, 발견 시 대사 출력 — 우선순위: 높음
- 힌트 시스템: 조합 가능한 방향 살짝 제시 — 우선순위: 중간

### 2차 (MVP 이후)
- 랜딩 페이지: 히어로 → 게임 소개 → 캐릭터 티저 → CTA
- 모에 일러스트 교체: 플레이스홀더 기하학 도형 → 직접 제작한 캐릭터 일러스트로 교체
- 유클리드 원론 2권~ 추가: DLC 방식으로 순차 추가
- 온라인 진행 저장: 로컬 → 서버 저장 (계정 시스템)

## 페이지 구조
| 페이지 | 경로 | 핵심 요소 | 디자인 여부 |
|--------|------|-----------|------------|
| 크래프팅 보드 | / | 카드 선택, 조합 슬롯, 결과 출력 | ❌ 토큰 기반 |
| 도감 | /collection | 전체 카드 목록, 언락 상태, 카드 상세 | ❌ 토큰 기반 |
| 랜딩 | /landing | 히어로 → 게임 소개 → 캐릭터 티저 → CTA | ❌ 2차 구현 |

## 기술 결정
- 스택: Next.js 15 / TypeScript / Tailwind / shadcn/ui / Zustand / Framer Motion
- 인증: MVP에서 제외 (로컬스토리지만 사용)
- DB: MVP에서 제외 (로컬스토리지)
- 배포: Vercel
- 조합 레시피 데이터: 정적 JSON 파일로 관리 (src/data/recipes.json)
- 카드 데이터: 정적 JSON 파일로 관리 (src/data/cards.json)

## 데이터 모델
- Card: { id, type(axiom/postulate/theorem), name, description, quote, imageUrl, unlockedBy[] }
- Recipe: { id, inputs: Card[], output: Card }
- GameState: { unlockedCards: id[], discoveredRecipes: id[] }

## 사용자 플로우
1. / 진입 → 공리 5개 + 공준 5개 카드 보유 상태로 시작
2. 카드 1~10장 선택 → 조합 시도 → 성공 시 새 정리 카드 언락 + 캐릭터 대사 출력
3. 언락된 정리 카드로 추가 조합 가능
4. /collection에서 전체 진행 상황 확인

## 제약 사항 / 합의 사항
- MVP는 유클리드 원론 1권만 (공리 5 + 공준 5 + 정리 48 = 총 58장)
- 캐릭터 일러스트는 MVP에서 플레이스홀더(기하학 도형)로 처리, 추후 직접 제작한 모에 일러스트로 교체
- 계정/인증 시스템은 MVP 제외
- 랜딩 페이지는 2차 구현
- 디자인 톤: 다크 배경, 골드 포인트 컬러, 기하학적 요소. Claude Code가 토큰 기반으로 자체 구현.
